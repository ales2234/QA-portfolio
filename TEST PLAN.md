| 1.INTRODUCTION This document outlines the testing strategy, scope, and schedule for evaluating the Amazon e-commerce platform. The objective of this test plan is to verify that the website’s core functionalities operate reliably and meet the required quality standards. The testing activities will concentrate on validating key user workflows, including account registration and authentication, product search and browsing, shopping cart management, and the checkout process.  |
| 2.OBJECTIVES The objectives of this testing effort are: Verify that the main features of the online store work correctly. Ensure users can complete purchases without errors. Validate system stability and usability across supported browsers.		 |
| 3.SCOPE 3.1 FEATURES TO BE TESTED User Registration User Login and Logout Product Listing Product Search Product Details Page Add Product to Cart Remove Product from Cart Update Product Quantity in Cart Checkout Process Order Confirmation 3.2 FEATURES NOT TO BE TESTED Third-party payment gateway functionality External shipping provider integrations Mobile native applications Performance and load testing  |
| 4.TEST STRATEGY The testing will include the following tests: Functional testing : validated that each feature works according to the specified requirements. UI testing : Ensure that interface elements such as buttons, forms, and navigation behave correctly and display properly. Exploratory testing : Perform unscripted testing to identify unexpected issues and usability problems.  |
| 5.TEST ENVIRONMENT Testing will be conducted in the following environments Operating system windows 11 Browsers Opera Google Chrome Microsoft Edge  |

| 7.ENTRY CRITERIA Testing will begin when: Test cases are prepared and reviewed. The testing environment is ready.  |
| 8.EXIT CRITERIA The testing will be considered completed when: All critical and high-priority defects are resolved. All planned test cases have been executed. The checkout process functions without critical failures.  |
